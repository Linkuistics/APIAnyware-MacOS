# Target: racket-oo

Racket OO bindings using the `tell` macro and class wrappers for idiomatic
Racket object-oriented interaction with macOS APIs.

## Before working here

1. Read `{{PROJECT}}/knowledge/targets/racket-oo.md` for target-wide learnings
2. Check `{{PROJECT}}/LLM_STATE/targets/racket-oo/backlog.md` for current progress

## For developers using the bindings

Read [`docs/developer-guide.md`](docs/developer-guide.md) — the
user-facing guide to writing Racket apps against macOS system APIs
via the generated bindings. Covers `tell` macro usage, dynamic
subclassing (`define-objc-subclass`), delegates and protocols,
threading and callback safety, app bundling, VM testing, and AppKit
widget quirks.

## Target structure

- `crate/` — Rust emitter crate (emit-racket-oo) — at
  `generation/crates/emit-racket-oo/`
- `runtime/` — Racket runtime modules
- `generated/` — output (regenerated by emitter, don't edit)
- `apps/` — sample app implementations
- `docs/` — user-facing developer documentation
- `tests/` — Racket-side smoke tests
- `test-results/` — GUIVisionVMDriver evidence (screenshots, reports)
- `lib/` — Swift dylib symlink
